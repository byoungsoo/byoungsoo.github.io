## [MCP(Model Context Protocol)](https://github.com/modelcontextprotocol)
MCP(Model Context Protocol)란?
MCP(Model Context Protocol)는 대규모 언어 모델(LLM) 애플리케이션과 외부 데이터 소스 및 도구 간의 원활한 통합을 가능하게 하는 개방형 프로토콜입니다. 이 프로토콜은 AI 모델이 필요한 컨텍스트와 연결되는 표준화된 방법을 제공합니다.

### [MCP 아키텍처](https://modelcontextprotocol.io/introduction#general-architecture)
MCP 아키텍처를 살펴보면 MCP 서버는 로컬에서 기동되며 MCP Client에서는 해당 하는 MCP 서버로 MCP 프로토콜을 통해 

![mcp_architecture.png](/assets/it/ml/agents/mcp_architecture.png)

#### MCP 구성요소
**MCP Hosts:** LLM 애플리케이션. Kiro, Claude, Cursor...
**MCP Clients:** 호스트에서 실행되는 플러그인. MCP 서버와 1:1로 연결되는 client.
**MCP Servers:** 도구, 컨텍스트, 프롬프트 등을 제공
**Local Data Sources:** `Your computer’s files, databases, and services` that MCP servers can securely access  
**Remote Services:** External systems available over the internet (e.g., through APIs) that MCP servers can connect to  

MCP 서버는 일반적으로 로컬 컴퓨터에서 실행되며 remote 서버에서 실행되는 경우 이러한 서버를 호스트형 hosted mcp server 또는 managed mcp server 라고 하지만 일반적이진 않다.  
로컬에서 실행되는 mcp 서버의 경우 표준입출력(stdio)을 통해 통신이 가능하며, hosted mcp server 와 통신할 때는 표준입출력 방식이 아닌 SSE(Server-Sent Event) 방식을 통해 구현되어야 한다. SSE는 서버에서 클라이언트로 실시간 데이터를 지속적으로 보내는 단방향 스트리밍 기술이다.

### MCP 설정방법

### [EKS MCP](https://docs.aws.amazon.com/amazonq/latest/qdeveloper-ug/command-line-mcp-understanding-config.html)
`mcp.json`  
```json
{
  "mcpServers": {
    "awslabs.eks-mcp-server": {
      "autoApprove": [],
      "disabled": false,
      "command": "uvx",
      "args": [
        "awslabs.eks-mcp-server@latest",
        "--allow-write",
        "--allow-sensitive-data-access"
      ],
      "env": {
        "FASTMCP_LOG_LEVEL": "ERROR"
      },
      "transportType": "stdio"
    }
  }
}
```

테스트는 kiro-cli 및 claude-code 를 중심으로 설명한다.  

**kiro-cli**
1. User level Configuration: `~/.kiro/settings/mcp.json` - Applies to all workspaces
2. Workspace Configuration: `<project-root>/.kiro/settings/mcp.json` - Specific to the current workspace

위 설정을 하고 나면 MCP 서버가 기동된것을 알 수 있다. mcp 서버는 uvx 커맨드를 통해 local 에서 기동된다.
