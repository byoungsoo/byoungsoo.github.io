### [MCP ВЋёьѓцьЁЇВ▓ў](https://modelcontextprotocol.io/introduction#general-architecture)
MCP ВЋёьѓцьЁЇВ▓ўвЦ╝ Вѓ┤ьј┤в│┤вЕ┤ MCP Вёюв▓ёвіћ вАюВ╗гВЌљВёю ЖИ░вЈЎвљўвЕ░ MCP ClientВЌљВёювіћ ьЋ┤вІ╣ ьЋўвіћ MCP Вёюв▓ёвАю MCP ьћёвАюьєаВйюВЮё ьєхьЋ┤ 

### MCP ЖхгВё▒ВџћВєї
- MCP Hosts: Programs like Claude Desktop, IDEs, or AI tools that want to access data through MCP  
- MCP Clients: Protocol clients that maintain 1:1 connections with servers  
- MCP Servers: Lightweight programs that each expose specific capabilities through the standardized Model Context Protocol  
- Local Data Sources: `Your computerРђЎs files, databases, and services` that MCP servers can securely access  
- Remote Services: External systems available over the internet (e.g., through APIs) that MCP servers can connect to  

1. ьўИВіцьіИ(Host)
- ВаЋВЮў: LLM ВЋаьћївдгВ╝ђВЮ┤ВЁўВю╝вАю, Вёюв▓ёвАювХђьё░ вЇ░ВЮ┤ьё░вЦ╝ в░ЏВЋё В▓ўвдгьЋе
- ВЌГьЋа:
  - ВЌгвЪг ьЂ┤вЮ╝ВЮ┤ВќИьіИ В┤ѕЖИ░ьЎћ в░Ј Ж┤ђвдг
  - ьЂ┤вЮ╝ВЮ┤ВќИьіИ-Вёюв▓ё ВѕўвфЁ ВБ╝ЖИ░ Ж┤ђвдг
  - ВѓгВџЕВъљ ЖХїьЋю вХђВЌг Ж▓░ВаЋ В▓ўвдг
  - ВЌгвЪг ьЂ┤вЮ╝ВЮ┤ВќИьіИ Ж░ё В╗еьЁЇВіцьіИ ьєхьЋЕ Ж┤ђвдг
- ВўѕВІю: Q developer chat, Cursor IDE вЊ▒

2. ьЂ┤вЮ╝ВЮ┤ВќИьіИ(Client)
- ВЌГьЋа:
  - вІеВЮ╝ Вёюв▓ёВЎђ 1:1 ВЃЂьЃю ВюаВДђ ВЌ░Ж▓░ Ж┤ђвдг
  - ьўИВіцьіИВЎђ Вёюв▓ё Ж░ё ВќЉв░ЕьќЦ вЕћВІюВДђ вЮ╝Вџ░ьїЁ
  - Вёюв▓ёВЮў вЈёЖхг, вдгВєїВіц, ьћёвАгьћёьіИ ьЁюьћївд┐ вЊ▒ ЖИ░віЦ Ж┤ђвдг
  - ьћёвАюьєаВйю в▓ёВаё в░Ј ЖИ░віЦ ьўИьЎўВё▒ ьўЉВЃЂ
  - Вёюв▓ё вдгВєїВіцВЌљ вїђьЋю ЖхгвЈЁ Ж┤ђвдг в░Ј в│ђЖ▓й ВЋївд╝ В▓ўвдг

3. Вёюв▓ё(Server)
- ВаЋВЮў: LLMВЌљ ВЎИвХђ вЇ░ВЮ┤ьё░ВЎђ В╗еьЁЇВіцьіИвЦ╝ ВаюЖ│хьЋўвіћ ЖИ░в│И ЖхгВё▒ ВџћВєї
- ВБ╝Вџћ ЖИ░віЦ:
    вЈёЖхг(Tools): LLMВЮ┤ ВЎИвХђ ВЋ▒Ж│╝ ВЃЂьўИВъЉВџЕьЋа Вѕў Въѕвіћ ВІцьќЅ Ж░ђвіЦьЋю ьЋеВѕў(Вўѕ: ьїїВЮ╝ вфЕвАЮ ВА░ьџї, Gmail/Slack API ьўИВХю)
    вдгВєїВіц(Resources): ьЁЇВіцьіИ ьїїВЮ╝, вАюЖиИ ьїїВЮ╝, DB ВіцьѓцвДѕ, ьїїВЮ╝ вѓ┤ВџЕ, Git ьъѕВіцьєавдг вЊ▒ LLMВЌљ ВХћЖ░ђ В╗еьЁЇВіцьіИ ВаюЖ│х
    ьћёвАгьћёьіИ ьЁюьћївд┐(Prompt Templates): ВќИВќ┤ вфевЇИ ВЃЂьўИВъЉВџЕВЮё ВЋѕвѓ┤ьЋўвіћ ВѓгВаё ВаЋВЮўвљю ьЁюьћївд┐ вўљвіћ ВДђВ╣е

вЈёЖхгвіћ вфевЇИВЮ┤ ВаюВќ┤ьЋўвЕ░, вдгВєїВіцВЎђ ьћёвАгьћёьіИвіћ ВѓгВџЕВъљЖ░ђ ВаюВќ┤ьЋЕвІѕвІц. вфевЇИВЮђ ВБ╝Вќ┤ВДё В╗еьЁЇВіцьіИВЌљ вћ░вЮ╝ вЈёЖхгвЦ╝ ВъљвЈЎВю╝вАю в░юЖ▓гьЋўЖ│а ьўИВХюьЋа Вѕў ВъѕВіхвІѕвІц.

Q chat ВЮё ьєхьЋ┤ EKS MCP вЦ╝ ВѓгВџЕьЋўвЕ┤ Hostвіћ qchat ВЮ┤ вљўЖ│а, Clientsвіћ MCP Clients, Вёюв▓ёвіћ MCP Вёюв▓ёЖ░ђ вљювІц.

### [EKS MCP](https://docs.aws.amazon.com/amazonq/latest/qdeveloper-ug/command-line-mcp-understanding-config.html)

1. Global Configuration: ~/.aws/amazonq/mcp.json - Applies to all workspaces
2. Workspace Configuration: .amazonq/mcp.json - Specific to the current workspace

`mcp.json`  
```json
{
  "mcpServers": {
    "awslabs.eks-mcp-server": {
      "autoApprove": [],
      "disabled": false,
      "command": "uvx",
      "args": [
        "awslabs.eks-mcp-server@latest",
        "--allow-write",
        "--allow-sensitive-data-access"
      ],
      "env": {
        "FASTMCP_LOG_LEVEL": "ERROR"
      },
      "transportType": "stdio"
    }
  }
}
```
