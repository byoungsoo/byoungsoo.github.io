# [Kiro]()  

## 1. [Steering]()  
What is steering?
Steering gives Kiro persistent knowledge about your workspace through markdown files. Instead of explaining your conventions in every chat, steering files ensure Kiro consistently follows your established patterns, libraries, and standards.



- Kiro에게 프로젝트에 대한 지속적인 지식을 마크다운 파일로 제공하는 기능입니다. 매번 채팅에서 컨벤션을 설명할 필요 없이, steering 파일에 한 번 작성하면 Kiro가 항상 참고합니다.

파일 위치:
- 워크스페이스: .kiro/steering/ (해당 프로젝트에만 적용)
- 글로벌: ~/.kiro/steering/ (모든 프로젝트에 적용)
- 충돌 시 워크스페이스가 우선

기본 제공 파일 3종:
- product.md — 제품 목적, 대상 사용자, 핵심 기능
- tech.md — 프레임워크, 라이브러리, 기술 스택
- structure.md — 파일 구조, 네이밍 컨벤션, 아키텍처

포함 모드 (inclusion):

| 모드 | 설명 |
|---|---|
| always (기본) | 모든 대화에 항상 포함 |
| fileMatch | 특정 파일 패턴 작업 시에만 포함 (예: components/**/*.tsx) |
| manual | #steering-file-name으로 직접 호출할 때만 포함 |
| auto | 요청 내용이 description과 매칭되면 자동 포함 |

## 2. [Skills]()  
## Skills (스킬)

재사용 가능한 지시 패키지입니다. [Agent Skills](https://agentskills.io) 오픈 표준을 따르며, 커뮤니티에서 가져오거나 공유할 수 있습니다.

핵심 동작 방식 — Progressive Disclosure:
1. 시작 시 스킬의 이름/설명만 로드 (가볍게)
2. 요청이 스킬 설명과 매칭되면 전체 지시사항 로드
3. 필요 시 스크립트/참조 파일까지 로드

폴더 구조:
my-skill/
├── SKILL.md           # 필수
├── scripts/           # 선택 - 실행 가능한 스크립트
├── references/        # 선택 - 참조 문서
└── assets/            # 선택 - 템플릿


파일 위치:
- 워크스페이스: .kiro/skills/
- 글로벌: ~/.kiro/skills/

사용 방법: 자동 활성화되거나, 채팅에서 /skill-name으로 직접 호출

## Steering vs Skills 차이

| | Steering | Skills |
|---|---|---|
| 목적 | 프로젝트 표준/컨벤션 | 재사용 가능한 워크플로우 |
| 포맷 | Kiro 전용 마크다운 | 오픈 표준 (Agent Skills) |
| 포함 방식 | always/fileMatch/manual/auto | 설명 매칭 시 자동 또는 / 명령 |
| 스크립트 | ❌ | ✅ scripts 폴더 지원 |
| 공유 | 팀 내 공유 가능 | GitHub 등에서 import 가능 |

간단히 말하면, Steering은 "우리 프로젝트는 이렇게 해"라는 규칙, Skills는 "이 작업은 이렇게 수행해"라는 절차입니다.
